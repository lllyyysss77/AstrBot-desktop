# Dashboard React 完整迁移校验 Checklist

> 基准：`dashboard/`（Vue）  
> 被检：`new-dashboard/`（React）  
> 目标：验证新版是否完整保留原 Dashboard 的页面结构、组件、API 契约、业务功能、交互反馈和桌面端行为。  
> 本清单不是“页面能打开”检查表；只有结构、数据、操作、异常状态和视觉行为均通过，才可判定对应功能完成迁移。

## 1. 使用方法

### 1.1 状态标记

- `[ ]`：未校验。
- `[x]`：已通过，且已填写证据。
- `[!]`：存在差异或缺陷，后附缺陷编号。
- `[-]`：确认不适用，后附批准人和原因。

每项建议追加以下信息：

```text
结果：通过 / 失败 / 不适用
环境：Web / Tauri，窗口尺寸，主题，语言
证据：截图、录屏、Network HAR、测试用例或日志路径
缺陷：BUG-xxx
备注：与旧版的差异及判断依据
```

### 1.2 校验顺序

1. 同时运行 Vue 和 React Dashboard，并连接同一版本后端。
2. 准备空数据、普通数据、大数据量、错误数据四组环境。
3. 先比较路由和页面结构，再比较 API，最后执行完整交互流程。
4. 对所有写操作核对请求方法、URL、参数、请求体和保存后的服务端结果。
5. 对所有弹窗、菜单和抽屉检查打开、确认、取消、遮罩、Esc 和焦点恢复。
6. 每个页面至少覆盖中文、英文、明暗主题和一个窄窗口。
7. 不以“React 实现方式不同”为失败理由；以用户可见行为和后端契约为准。

### 1.3 已批准的差异

- [ ] Chat Markdown 和流式内容改用 Streamdown；不要求复刻旧 Markdown 渲染器内部实现，仅校验最终内容、代码块、链接、引用、滚动和流式稳定性。
- [ ] “配置文件”编辑弹窗仅保留 JSON 编写模式，不要求迁移旧版 Form/JSON 切换。
- [ ] Vue/Vuetify 与 React/headless 组件的 DOM 层级和类名不要求一致。
- [ ] 已注释、未启用的旧 Alkaid 子页面不属于迁移范围；`/alkaid/knowledge-base` 兼容入口仍须可用。
- [ ] 任何新增差异必须在本节登记，并由产品/维护者确认，不能由实现者自行视为“不适用”。

## 2. 验收环境与数据矩阵

### 2.1 运行环境

- [ ] 使用本机 NVM 选择项目要求的 Node 版本后，依赖安装、开发启动、类型检查和构建均通过。
- [ ] `pnpm run dev:dashboard:new` 可独立启动新版，不依赖旧 Dashboard 运行时。
- [ ] `pnpm run typecheck:dashboard:new` 通过。
- [ ] 新版单元测试通过。
- [ ] `pnpm run prepare:webui:new` 生成可被桌面端加载的资源。
- [ ] 浏览器 Hash 路由与 Tauri 内嵌页面均完成验证。
- [ ] 页面刷新、直接输入深链接、浏览器前进/后退均正常。

### 2.2 窗口与显示

- [ ] 桌面大窗口（建议 1920×1080）。
- [ ] 桌面中窗口（建议 1366×768）。
- [ ] 接近最小窗口尺寸。
- [ ] 浏览器缩放 80%、100%、125%。
- [ ] 长中文、长英文、长插件名、长模型名不会遮挡关键操作。
- [ ] 页面只有预期的滚动容器，不出现正文和外层窗口争抢滚动。
- [ ] 固定按钮、浮动按钮和弹窗底栏以可视窗口为基准，不落到长页面最底部。

### 2.3 状态矩阵

- [ ] 首次安装/无配置数据。
- [ ] 普通数据量。
- [ ] 大数据量和多页数据。
- [ ] 慢请求、并发请求和重复点击。
- [ ] 401、403、404、409、422、500、网络断开和请求超时。
- [ ] 空数组、缺失可选字段、`null`、历史版本数据均不会造成白屏。
- [ ] 加载、空态、错误态、局部刷新和重试状态均与旧版语义一致。

### 3.2 鉴权

- [ ] 除登录和初始化页外，受保护路由未登录时统一跳转登录。
- [ ] 初始化状态正确决定进入 `/auth/setup` 还是 `/auth/login`。
- [ ] 登录支持用户名、密码、TOTP 和恢复码流程。
- [ ] 初始化支持管理员创建、TOTP 设置、恢复码展示与确认。
- [ ] 登录成功恢复原目标地址，而非固定丢回欢迎页。
- [ ] Token 的存储键、Authorization 格式和旧版一致。
- [ ] 401 会清理失效会话并跳转登录；不会产生无限重试。
- [ ] 退出登录清理敏感状态、缓存和流式连接。

### 3.3 FullLayout

- [ ] 顶栏 Logo、菜单按钮、Chat 入口和更多菜单的位置、尺寸、点击区域与旧版一致。
- [ ] 侧栏主项顺序一致：欢迎、机器人、模型提供商、配置文件、插件、知识库、人格设置、更多功能、设置。
- [ ] “更多功能”展开项及顺序一致。
- [ ] 插件入口的子导航或页内 Tab 能覆盖旧版已安装插件、市场、管理行为、MCP、Skills 入口。
- [ ] 当前路由高亮、父级展开和深链接后的高亮正确。
- [ ] 侧栏滚动与底部设置项固定行为一致。
- [ ] 侧栏折叠、移动端抽屉、遮罩关闭和窗口变化行为正确。
- [ ] 自定义侧栏顺序、隐藏项等偏好可保存并恢复。
- [ ] 页面主内容高度随窗口变化，不因固定像素高度产生外层多余滚动。
- [ ] 全局加载条、Toast、确认框、错误边界可用且不会被弹窗遮挡。

## 4. 全局组件与基础能力

### 4.1 主题、国际化和资源

- [ ] 明亮、暗色、跟随系统三种主题可切换并持久化。
- [ ] 刷新时首屏不会先闪烁错误主题。
- [ ] 中文、英文及旧版支持的其他语言可切换并持久化。
- [ ] API 需要语言参数或请求头时与当前语言同步。
- [ ] 插件元数据、配置描述和后端 i18n 内容按当前语言回退。
- [ ] Logo、平台图标、插件图标、空态图、Emoji 和字体资源加载正常。
- [ ] 不存在未翻译 key、`undefined`、`[object Object]` 或开发占位文本。

### 4.2 通用弹窗、菜单和表单

- [ ] Dialog 打开后焦点进入弹窗，Tab 不逃逸到背景。
- [ ] Esc、遮罩、取消按钮是否关闭与旧版各场景一致。
- [ ] 关闭后焦点返回触发按钮。
- [ ] 菜单锚定正确，不被滚动容器裁剪，点击外部可关闭。
- [ ] 确认按钮在提交中禁用，避免重复写入。
- [ ] 删除、覆盖、重启、卸载等危险操作有二次确认。
- [ ] 文本框、选择器、开关、复选框、文件选择和拖放均支持键盘。
- [ ] 校验错误与字段关联，保存失败不丢失用户输入。
- [ ] 长列表选择器支持滚动、搜索、空态和完整信息展示。

### 4.3 动态配置渲染

- [ ] 字符串、数字、布尔、枚举、列表、对象、模板、文件等配置类型完整。
- [ ] 必填、默认值、范围、正则、条件显示和只读属性与 schema 一致。
- [ ] 嵌套对象和数组的新增、删除、排序不会改变无关字段。
- [ ] 密码/密钥字段不会在日志、错误或 UI 中意外明文回显。
- [ ] Provider、Persona、知识库、插件集合等专用选择器语义一致。
- [ ] JSON 编辑器语法高亮、格式化、错误定位、滚动和大文本性能可接受。
- [ ] 未修改离开页面时不提示；有修改时导航、刷新和关闭窗口均提示。
- [ ] 受保护保存正确触发二次验证流程。

### 4.4 HTTP 与 OpenAPI

- [ ] API 基址统一为旧版契约，不在页面中散落硬编码域名。
- [ ] OpenAPI 生成客户端和手写请求使用相同鉴权、错误解析与响应解包。
- [ ] GET 查询数组、布尔值、空值和特殊字符编码正确。
- [ ] 路径参数（插件名、UMO、文件名等）正确 URL 编码。
- [ ] JSON、FormData、Blob、下载文件、SSE 和 WebSocket 分别使用正确处理方式。
- [ ] 错误提示优先展示服务端业务消息，同时保留可诊断信息。
- [ ] 快速切换筛选或路由时，旧请求结果不会覆盖新状态。
- [ ] 组件卸载会取消轮询、定时器、事件监听和长连接。
- [ ] 写入成功后采用与旧版一致的局部更新或重新拉取策略。

## 30. API 契约审计方法

每个页面除功能勾选外，再执行一次 Network/API 差分审计：

- [ ] 旧版和新版在同一初始数据下执行同一操作并导出 HAR。
- [ ] 请求方法一致。
- [ ] URL 和路径参数一致。
- [ ] Query 参数名称、默认值、空值省略规则和数组编码一致。
- [ ] 请求体字段、类型、默认值及未知字段保留策略一致。
- [ ] Content-Type、Authorization、语言和 2FA Header 一致。
- [ ] 成功响应的解包、缓存和列表刷新策略一致。
- [ ] 204、空 Body、Blob、流式响应均不会被按普通 JSON 错误解析。
- [ ] 业务失败、HTTP 失败和网络失败的提示语义一致。
- [ ] 乐观更新失败时回滚。
- [ ] 删除/更新接口不会因重复点击执行两次。
- [ ] 所有轮询都有终止条件、间隔和页面卸载清理。

建议记录格式：

| --- | --- | --- | --- | --- | --- |
| 示例：插件 | 禁用插件 | `POST ...` | `POST ...` | 失败均回滚开关 | 通过 |
